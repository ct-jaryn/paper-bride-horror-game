# 阴阳簿 · Paper Bride Horror Game

> 一个多故事、共享世界观的 **中式文字恐怖游戏集**。  
> 每一卷都是独立的民俗恐怖故事，发生在同一个「阴阳交错」的世界里，共享相同的民俗符号、诅咒规则与因果逻辑。
>
> 🎮 **在线预览**：https://ct-jaryn.github.io/paper-bride-horror-game/  
> 📦 **源码仓库**：https://github.com/ct-jaryn/paper-bride-horror-game

---

## 简介

《阴阳簿》记录人与鬼之间未了之事。簿中的每一页，都是一个被执念困住的灵魂。它们或因冤而死，或因爱而不舍，或因罪而难以超生。只有"有缘人"才能翻开这些页面，进入它们的故事。

你，就是这本簿子的最新记录者。

## 已收录故事

### 卷一 · 《回门》

**主题**：冥婚 · 纸人 · 山村罪孽

你收到祖母病逝的家书，回到三十年未归的山村。村里正在筹备一场"喜丧"，棺材上贴着倒挂的"囍"字。原来三十年前，少女秀兰被活埋成了一场冥婚的新娘，而你的脸，和她未婚夫一模一样。

**核心恐怖**：纸人转头、棺材异响、红绳勒进皮肉、井下红棺

### 卷二 · 《守夜》

**主题**：诈尸 · 殡仪馆 · 未唱完的戏

你是县殡仪馆的守灵人。这一夜，七号厅的无名女尸睁开了眼睛。她穿着大红戏服，指甲缝里全是河泥。她不想害人，她只想唱完那出没唱完的《牡丹亭》。

**核心恐怖**：睁眼尸体、镜中倒影、黑水、合唱戏班

### 卷四 · 《替身》

**主题**：纸扎 · 挡灾 · 身份置换

你五年未归，回到老家时发现院子里站着一个纸扎的"你"。父母说那是为了给你挡病，可纸人越扎越像，最后连母亲都分不清谁是真的你。

**核心恐怖**：纸人睁眼、身份替换、被自己的脸追赶、烧成灰烬仍不灭

### 卷五 · 《河娘》

**主题**：溺亡 · 红裳 · 河边唱歌的女人

母亲的葬礼在雨天举行。她不是病死，而是自己走进河里的。村里人说，她是被"河娘"叫走的。你小时候也曾差点被河娘拉下水，是母亲把你抢回来的。

**核心恐怖**：水下歌声、河面倒影、三十年前的承诺、母女身份的倒置

### 卷六 · 《狐嫁》

**主题**：狐妖 · 人婚 · 长生

你是山脚下的猎户之子周远。青丘山狐女修炼千年，只差一场人婚。她选中了你，许你长生不死。拒绝她，全村人替你答应。

**核心恐怖**：红轿无夫、狐瞳、人妖殊途、长生背后的代价

### 卷七 · 《赶尸》

**主题**：湘西 · 雨夜 · 三具尸体

你是湘西最年轻的赶尸匠田小七。这一单要送三具尸体翻山越岭，可其中一具在对你笑。雨夜山路，铜铃停了，而尸体还站着。

**核心恐怖**：尸笑、雨夜山路、铜铃、尸变、赶尸匠的规矩

## 中式恐怖核心元素

### 视觉符号
- **红与白**：红烛、红嫁衣、红盖头 vs 白幡、白纸人、白灯笼
- **纸扎**：纸人、纸马、纸新娘，会转头、会笑
- **镜子**：照不出自己，或照出多一个人
- **水与井**：溺亡、倒影、井下有人
- **棺材与灵堂**：指甲刮擦声、挪动痕迹
- **戏台与戏曲**：唱不完的戏、无脸戏子、戏服长在身上
- **农历与时辰**：子时、丑时、寅时，鬼门开关

### 听觉设计
- **二胡**：单音、走调、时断时续
- **风声**：穿过破窗、纸钱沙沙
- **脚步声**：木地板吱呀，但屋内只有你
- **低语**：女声哼唱旧戏曲，越来越近
- **突发音**：铜锣、摔碗、婴儿啼哭、棺材盖落地

### 叙事节奏
- **诡异但不吓人**：前 30% 营造"不对劲"感
- **小惊吓**：纸人转头、门自己开、镜中倒影
- **大恐怖**：追逐、真相揭露、玩家必须做出道德选择
- **绝望反转**：你以为逃出去了，其实……

## 游戏系统

### 1. 理智值（SAN）
- 范围 0–100
- 看到恐怖事件下降
- 过低会产生幻觉，文本中出现真实与虚假混合
- 归零：角色精神崩溃，进入特殊结局

### 2. 阴气值（YIN）
- 范围 0–100
- 在阴气重的地方停留、做错选择会上升
- 阴气高：能听到亡者低语，看到一些隐藏线索
- 阴气过高：会被附体，直接死亡

### 3. 时辰系统
- 戌时（19-21）：入村/开场
- 亥时（21-23）：守灵/铺垫
- 子时（23-01）：鬼门开
- 丑时（01-03）：阴气最盛
- 寅时（03-05）：生死一线
- 卯时（05-07）：天亮或永夜

每个选择推进 15–60 分钟。

### 4. 物品系统
- 铜镜/铜镜碎片：照出真实
- 护身符：辟邪挡灾
- 打火机：烧纸、照明
- 云袖的银钗：关键证物
- 秀兰的嫁衣：超度道具
- 小生戏服：扮演柳梦梅

### 5. 关键状态
- `knowsTruth`：是否知道真相
- `xiulanMercy`：秀兰的好感/宽恕
- `knowsDesire`：是否知道亡者心愿
- `promised`：是否承诺帮助

### 6. 成就系统

游戏内置 18+ 项成就，分为单卷成就、关键结局成就与跨卷联动成就：

- **单卷成就**：完成《回门》《守夜》《戏台》《替身》《河娘》《狐嫁》《赶尸》
- **关键结局成就**：如「慈悲为怀」「纸人替身」「三十年之歌」「狐婿长生」「夜路走多」
- **跨卷成就**：需要在不同故事间触发联动条件，例如
  - 「周家迷局」：在《狐嫁》中身为周远，又从《赶尸》抵达青石镇周家
  - 「青石镇往事」：在《赶尸》中抵达青石镇，揭开与其他故事共享的世界一角
  - 「真相追寻者」：在三卷不同故事中触及核心真相

成就数据统一保存在 `localStorage` 的 `huimen_save_v2` 中（旧版 `huimen_achievements` 会在首次启动时自动迁移）。

### 7. 跨卷联动（阴阳世界）

七卷故事共享同一个世界观，部分关键事件会写入**跨卷世界状态**（`huimen_global_flags`）：

- 玩家在《赶尸》中抵达周家（`hujia_to_zhou`）或青石镇（`ganshi_to_qingshi`）
- 玩家在《狐嫁》中身为周远，与《赶尸》《回门》《戏台》中的周家形成暗线呼应
- 多卷中收集到的真相类 flag 可解锁「真相追寻者」等跨卷成就

当你开始一卷新故事时，引擎会自动把已获得的全局 flag 合并进当前故事状态，使后续故事出现因前置经历而触发的额外可能。

### 8. 音效系统

基于 Web Audio API 程序化生成，无需外部音频文件。支持：

- 风声环境背景音
- 按钮点击、翻页沙沙声
- 纸人惊吓、铜钱铃铛、结局低语
- 静音开关与音量记忆

### 9. CG / 插画节点

场景数据可配置 `cg` 字段，进入场景时自动全屏展示插画。无真实图片时以中式恐怖风格占位。已解锁 CG 收入「残卷图鉴」，可在标题画面或故事选择页查看。

### 10. 移动端适配

针对手机竖屏优化：增大触摸目标、调整字体与状态栏、故事卡片与成就面板单列显示、选择按钮长文本自动换行。

### 11. 隐藏结局与跨卷彩蛋

《回门》《戏台》《狐嫁》《赶尸》中新增了由全局 flag 触发的隐藏结局（`hidden_` 前缀），鼓励玩家在多卷之间探索联动。

新增的 `js/engine/storyExtensions.js` 彩蛋系统进一步在各卷中埋下暗线：当玩家在其他故事中达成特定条件后，当前故事的关键场景会出现额外的隐藏选项。这些彩蛋以“似曾相识的旧报纸”“路人低语”“衣角纹样”等模糊形式出现，不会破坏每卷独立成篇的体验，却让多周目玩家能察觉到七卷之间的隐秘联系。

### 12. 选择记录与复盘

游戏内“记”按钮可随时打开「命簿记录」浮层，查看本局每一次选择的：

- 所在场景
- 选择文本
- 理智 / 阴气 / 时辰变化
- 获得的物品或触发的 flag

记录面板新增：

- **导出**：一键复制当前故事的全部选择记录到剪贴板（不支持剪贴板时弹出文本框）。
- **复盘**：根据 `history` 与 `choiceLog` 重建本局场景路径，以时间线形式回顾“在哪一场景做出了什么选择”。

「阴阳历程」面板则汇总七卷故事的游玩进度、已选择次数与已达成结局。

### 13. 虚拟货币「阴钱」与阴市

- 货币名称：阴钱
- 获取途径：抵达结局、完成故事、解锁成就
- 用途：购买「通灵符」获得场景提示、解锁「窥梦图」CG、死亡结局时「借命还阳」
- 商店：标题画面与故事选择页均可进入「阴市」
- 充值：提供标注为「模拟充值（演示）」的测试入口，用于功能演示，非真实支付

### 14. 核心命名空间 `Huimen`

为避免全局变量过多导致的隐式耦合，`js/engine/namespace.js` 将所有引擎状态与公共 API 集中在 `window.Huimen` 下：

- `Huimen.CurrentStory`、`Huimen.StoryData`、`Huimen.Endings`
- `Huimen.GameState`、`Huimen.GlobalFlags`
- `Huimen.GameEvents`（事件总线）
- `Huimen.ToastManager`（统一提示）
- `Huimen.StoryManifest`、`Huimen.loadStory`、`Huimen.showScreen`

扩展模块（`sound.js`、`cg.js`、`currency.js`）统一通过 `Huimen` 访问引擎状态，降低模块间耦合。

## 多结局设计

每个故事都有多个结局，包括：

- **真结局**：化解执念，超度亡魂
- **普通结局**：暂时平息，但未根本解决
- **险胜结局**：勉强逃生，但代价惨重
- **死亡结局**：被鬼附身、成为祭品、永远困住
- **开放式结局**：看似逃脱，实则诅咒延续

## 如何添加新故事

1. 在 `stories/` 目录下新建 `story_yourname.js`
2. 导出 `StoryData` 与 `Endings`（例如：`export { StoryData, Endings };`）
3. 在 `stories/manifest.js` 中注册新故事
4. 引擎会自动按需动态加载并支持存档

故事数据结构：

```javascript
export const StoryData = {
    prologue: {
        text: `你的剧情文本...`,
        effects: {
            sanity: -5,
            yin: 10,
            visual: 'flicker'
        },
        choices: [
            {
                text: '选项A',
                next: 'scene_a'
            },
            {
                text: '选项B（危险）',
                next: 'scene_b',
                danger: true,
                effects: { sanity: -10 }
            }
        ]
    },
    // ... 更多场景
};

export const Endings = {
    myEnding: {
        title: '结局标题',
        text: '结局文本...'
    }
};
```

## 扩展建议

1. **多语言支持**：保留中文韵味，添加英文翻译
2. **云存档**：将存档、成就、货币同步到服务端，并做服务端校验（当前纯前端，本地数据可被篡改）
3. **服务端校验**：若接入真实支付、排行榜或云存档，必须由服务端校验关键数据
